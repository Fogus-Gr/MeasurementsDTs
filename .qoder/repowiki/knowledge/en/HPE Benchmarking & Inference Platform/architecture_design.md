- **Inference Core**: `main.py` acts as the CLI entry point, dispatching to backend implementations (`alphapose_hpe.py`, `movenet_hpe.py`, `openvino_base_hpe.py`) that inherit from the abstract `BaseHPE` class. `BaseHPE` manages input routing (image/video/RTSP), frame preprocessing (padding/resizing), and COCO-format output serialization.
- **Benchmarking Rigs**: The system is organized into Docker-based experiment rigs (`ffmpeg_hpe/`, `monitor_hpe/`). The primary rig (`ffmpeg_hpe/`) orchestrates a multi-service stack via `docker-compose.yaml`: an RTSP broker (MediaMTX), a video producer (FFmpeg/NVENC), the HPE inference container, and monitoring sidecars for GPU metrics (`gpu-metrics`), CPU/memory profiling (`perf_monitor`), and kernel-level RX byte tracing (`bcc-tracer`).
- **Observability Split**: Network telemetry is split by direction due to kernel constraints: TX bytes are traced via `bpftrace` syscall hooks in the `perf_monitor` container, while RX bytes are captured by a BCC/eBPF socket filter in the `bcc-tracer` container, which shares the HPE network namespace.
- **Configuration Flow**: Experiment scripts (`run_experiment.sh`) dynamically detect hardware resources (vCPUs, GPU availability) and inject environment variables (e.g., `OV_THREADS`, `HPE_DEVICE`) into the compose stack, overriding static `.env` defaults to ensure optimal resource isolation between inference and telemetry services.
