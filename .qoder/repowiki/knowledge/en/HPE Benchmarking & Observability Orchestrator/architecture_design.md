- **Unified Experiment Rigs**: Two primary Docker Compose stacks (`ffmpeg_hpe` and `monitor_hpe`) serve as the execution backbone, orchestrating HPE inference containers alongside sidecars for GPU metrics (DCGM/nvidia-smi), CPU profiling, and network tracing (BCC/eBPF).
- **Dynamic Resource Injection**: Root-level `Makefile` targets and shell scripts (`run_experiment.sh`) act as the control plane, auto-detecting host vCPUs to dynamically calculate and inject resource limits (CPU shares, memory caps, OpenVINO threads) into the compose environment at runtime.
- **Shared Telemetry Contracts**: All rigs enforce a consistent observability pattern where sidecar containers share the HPE service's PID or network namespace to capture high-fidelity, process-specific metrics (RX/TX bytes, RSS, CPU%) without modifying the inference code.
- **Hardware-Aware Gating**: The orchestrator implements runtime gating for GPU dependencies, conditionally enabling the NVIDIA container runtime only for heavy models (AlphaPose/OpenPose) while isolating lightweight models (MoveNet/OpenVINO) to CPU-only execution paths.
