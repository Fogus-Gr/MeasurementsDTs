- Entry point: `run_experiment.sh` orchestrates the full lifecycle — validates inputs, auto-detects vCPUs, resolves per-method GPU/CPU tuning, starts services via `docker-compose.yaml`, monitors completion, and collects artifacts.
- Service topology (defined in `docker-compose.yaml`): `rtsp-broker` (MediaMTX relay), `streamer` (FFmpeg NVENC encoder pushing to RTSP), `hpe` (Python HPE consumer reading from RTSP), `gpu-metrics` (nvidia-smi polling), `perf_monitor` (host-PID bpftrace/ps metrics), and `bcc-tracer` (BCC socket-filter RX/TX byte accounting).
- Dynamic resource allocation: `run_experiment.sh` exports `HPE_CPU_LIMIT`, `HPE_MEMORY_LIMIT`, `OV_THREADS`, etc., which are substituted into the `hpe` service definition; sidecars are capped aggressively to avoid contaminating HPE measurements.
- Network tracing boundary: `bpftrace-tracer/entrypoint.sh` detects the ephemeral HPE-side TCP port via `ss`, then runs `bcc_rx_bytes.py` (foreground, socket filter on eth0) and `bcc_tx_bytes.py` (background, sys_enter_sendto) to produce CSV traces.
- GPU/runtime gating: `run_experiment.sh` sets `HPE_RUNTIME` (runc vs nvidia) and `HPE_NVIDIA_VISIBLE_DEVICES` so CPU-only methods run without a GPU driver while the streamer retains GPU access for NVENC.
